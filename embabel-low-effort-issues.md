## Effort scale

| Effort | Meaning | Typical session |
|--------|---------|------------------|
| **S** | Small — docs, typos, tiny refactor, single-file fix | 15–30 min |
| **M** | Medium — one feature, small API change, tests for one class | 30–60 min |
| **L** | Low-but-larger — multi-file, needs design; still “low effort” relative to core work | 1–2 hr |

Only add issues you consider **S** or **M** here so they’re realistic buffer work.

## How to use in planning

1. **Weekly** — When filling your [monthly plan](plans/), add a line: “If remaining $: 1–2 from [embabel-low-effort-issues.md](embabel-low-effort-issues.md).”
2. **Mid-month** — Run `./scripts/usage-remaining.sh`; if you’re under pace, schedule 1–2 sessions from this list to use the rest of your $400.
3. **Session** — Open the issue, clone repo if needed, then in Cursor: “Implement/fix [issue title]. Context: [link].” Complete and close the issue.
